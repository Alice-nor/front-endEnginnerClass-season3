2021/07/02 Networks Management

Basics of Web Design

交換器的雙工（Duplex）

  • 全雙工表示同一時間可以送出資料也可以接收資料，而不會產生碰撞（switch，也可轉成半雙工）
  • 半雙工表示同一時間只能送出資料或接收資料，並且會有碰撞（hub）

交換器轉送方法：

  • 儲存轉送（store-and-forward） - 較可靠，但速度慢。交換器收到資料時會將資料存在緩衝區（RAM），直到確認資料訊框的完整性再送往目的地。
  • 截穿交換（cut-through） - 比較快，但資料沒有檢查。在收到資料資料訊框時立即處理資料,交換器不執行任何錯誤檢查。

某些交換器可設定為連接埠先執行截穿交換,當達到使用者定義的錯誤比例門檻（10%）時,這些連接埠自動切換為儲存轉送,當錯誤率低於該門檻時,連接埠自動恢復到截穿交換。


fragment-free 模式

  • 在此模式下交換器會先讀取 Frame 的前 64 個 bytes 並做檢查
  • 前 64 個 bytes 有錯誤,交換器將會把 Frame 丟棄
  • Fragment-Free 就兼具 store-and-forward 和 cut-though 的優點。

廣播資料(Broadcast Message)

  • 廣播資料就是要將資料送給所有電腦（不做 mac 檢查）
  • 交換器、集線器接收到廣播資料就會轉送所有電腦
  • 若用廣播資料傳送病毒，所有電腦都會收到
  • 理解路由器 Router 前要先理解廣播資料

不同廣播區域：

  • 不同廣播區域下的電腦是無法接收到對方送出的廣播資料
  • 若將交換器與集線器串接，只有一個廣播區域
  • 不管串接多少台 swich 或 hub，都屬於同一個廣播區域，所以 switch 與 hub 都無法阻擋廣播資料

路由器（router）

  • 有阻擋廣播資料的功能
  • 有切割廣播區域的功能
  • 主要設計不是用來接電腦，是用來切割與規劃網路
  • 廣播區域當作一個網路區域或簡稱網路，廣播區域 = 網路區域
  • 只有路由器才能將網路區域連接起來，連接起來的網路稱為互聯網（inter network / internet）
  • 要劃分網路區其主要目的是要定位電腦的所在位置，因此要找某一台電腦，要先判斷此電腦在哪一個網路區域下

網路區域編號（ip）

  • 前三碼為區域編號，後一碼為電腦編號（ex:192.168.10.0）
  • 每個網路區域需要一個編號，編號有一定格式稱為 ip
  • 網路區域的實際地理位置，規劃網路的人員知道

路由器為路由表來作為繞送查詢使用。 路由表主要內容是網路區域 ip 的位址對應 port（switch 則是在 mac table 的 mac 位址對應到 port）


應用層，常見到的應用層協定有：

  1. 網域名稱服務協定 DNS：用於網際網路網域名稱解析（domain.name）
  2. 超文字傳輸協定 HTTP / HTTPS（加密）：用於傳輸組成全球資訊網網頁（www）的檔案
  3. 檔案傳輸協定 FTP：用於在系統之間相互傳輸檔案
  4. 檔案傳輸協定 TFTP：簡易版的 FTP，不需登入帳號

應用協定的編號：每種應用層協定都會有一個編號，編號稱為 port no.。以下為範例 port no.

  • DNS / 53
  • HTTP / 80
  • HTTPs / 443
  • 遠端登入協定 Telnet / 23
  • 簡易檔案傳輸協定 TFTP / 69

傳輸層 - TCP 方式是一種可靠性的傳輸協定（與 UDP 相反）

常見 TCP 應用程式：

  • Web 瀏覽器
  • 電子郵件
  • 檔案傳輸協定（FTP）
  • 遠端登入協定 Telnet

常見 DCP 應用程式：

  • 網域名稱系統（DNS）
  • 網管軟體協定（SNMP）
  • 視訊串流
  • 簡易檔案傳輸協定（TFTP）
